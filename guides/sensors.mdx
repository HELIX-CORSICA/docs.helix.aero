# Sensor Catalog

Helix Flight Planner ships with a rich catalog of **sensors** (cameras, LiDAR, pushbroom) and **aircraft** (helicopters, airplanes, drones). This guide explains how the catalog works from the user's point of view — searching, filtering, switching between grid and table view, and creating your own custom entries.

For the mission-time side (how a selected sensor drives strip spacing and GSD) see the [Mission Planning guide](/guides/mission-planning).

## Opening the Catalog

Open the **Catalog** from the left navigation. The catalog has two tabs:

- **Sensors** — cameras, LiDAR and pushbroom systems.
- **Aircraft** — helicopters, airplanes and drones.

A small sync indicator on the header shows whether your catalog is up to date. It refreshes automatically when you log in and whenever sync runs.

## Searching and Filtering

Both tabs share the same control bar at the top:

- **Search field** — matches on sensor or aircraft name, manufacturer and model. The query is cleared when you switch tabs.
- **Type chips** — for the sensor tab: **All**, **Camera**, **LiDAR**, **Pushbroom**. For the aircraft tab: **All**, **Helicoptere**, **Avion**, **Drone**.
- **View toggle** — switches between grid and table view.

Filters and search combine — for example, `Search: "sony" + Type: Camera` narrows the grid to Sony cameras only.

## Grid View vs Table View

The view toggle (top right of the catalog) switches between two layouts:

<CardGroup cols={2}>
  <Card title="Grid view" icon="grip">
    A responsive grid of cards — each card shows a photo of the sensor or aircraft, its name, manufacturer and the key specs. Best for quick visual browsing.
  </Card>
  <Card title="Table view" icon="table">
    A dense sortable table — click any column header to sort ascending / descending. Best when you want to compare many items on a single spec (for example, focal length or FOV).
  </Card>
</CardGroup>

The view preference is remembered between sessions.

## Sensor Details

Click any sensor card (or row) to open the **Sensor Details** dialog. It shows the full specifications captured from the catalog:

### Camera sensors

| Specification | Example |
|---------------|---------|
| Focal length | 35 mm |
| Sensor width / height | 35.9 x 24 mm |
| Pixel size | 4.51 µm |
| Image resolution | 7952 x 5304 px |
| Shutter type | Global / rolling |
| Recommended forward overlap | 80% |

### LiDAR sensors

| Specification | Example |
|---------------|---------|
| FOV | 120 deg |
| Pulse repetition rate | 600 kHz |
| Scan rate | 100 Hz |
| Beam divergence | 0.5 mrad |
| Maximum range | 500 m |
| Multi-return | Up to 5 returns |

### Pushbroom sensors

| Specification | Example |
|---------------|---------|
| Line length | 12 000 px |
| Pixel size | 6.5 µm |
| FOV | 40 deg |
| Acquisition frequency | 400 Hz |

The dialog also offers a **Use for new mission** shortcut that opens the mission wizard prefilled with the sensor selected.

## Standard vs Custom Sensors

Sensors and aircraft come in two flavors:

- **Standard** — curated by Helix Aero. These cannot be edited or deleted locally; if a property is wrong, please contact support. Standard entries are shared across all users in your organization and update automatically.
- **Custom** — created by you. They appear alongside standard entries with a small **Custom** badge and are scoped to your account. They can be edited or deleted at any time.

<Warning>
  Deleting a custom sensor does not affect missions already created with it. Missions keep their own snapshot of the sensor specs at the moment they were created.
</Warning>

## Creating a Custom Sensor

Click **New Sensor** at the top of the catalog (or the floating action button in the sensor panel of a mission). A form dialog opens:

<Steps>
  <Step title="Pick the sensor type">
    Camera, LiDAR or Pushbroom.
  </Step>
  <Step title="Fill in identity">
    Name, manufacturer and model.
  </Step>
  <Step title="Fill in the technical specs">
    The fields shown depend on the type (see the specification tables above). Required fields are marked; the app will not let you save until they are filled.
  </Step>
  <Step title="Save">
    The sensor is stored locally and synced to your account at the next sync cycle. It is immediately available in the mission wizard.
  </Step>
</Steps>

<Tip>
  For cameras, if you know the pixel size and image resolution, the sensor width and height are derived automatically. You can also type the sensor dimensions directly and pixel size is computed from them.
</Tip>

## Creating a Custom Aircraft

The aircraft tab supports the same flow. Custom aircraft let you define default altitude, cruise speed and overlap values that will be used to seed new missions. The mission wizard's aircraft step lists custom aircraft next to the standard catalog.

## Editing and Deleting

For custom entries only, click the small menu icon on the card (or row) to access **Edit** and **Delete**. Standard catalog entries show a lock icon instead — to propose changes or fixes, contact Helix Aero support.
