# Export Formats

Helix Flight Planner can export a completed mission plan in many formats. Open the **Export** menu from the top toolbar of the mission view (or press `Ctrl+E`) to pick one. The menu is grouped into:

- **Flight plan** — geometric / data formats for external tools (KML, KMZ, GPX, CSV, GeoJSON, XLSX).
- **Client view** — an HTML deliverable for non-technical stakeholders.
- **Drone Waypoints** — a CSV profile tuned for drone controllers.
- **PDF report** — a printable multi-page mission report.
- **Native Helix file** — an opaque container you load locally into the Helix FMS.

## KML — Google Earth and GIS

**File extension:** `.kml`

KML (Keyhole Markup Language) is the standard geographic format used by Google Earth and most GIS tools.

**What's included:**
- AOI polygon boundary
- Flight strip centerlines as `LineString` placemarks
- Camera footprint polygons for each frame
- Strip labels with heading and sequence number

**When to use:**
- Sharing the mission with the pilot for pre-flight review in Google Earth
- Importing into QGIS, ArcGIS or similar for further analysis
- Submitting a visual flight area to airspace authorities

## KMZ — Zipped KML

**File extension:** `.kmz`

A zipped KML archive with embedded styles and raster overlays. KMZ travels better than KML when the export includes styled icons or QC heatmaps.

**When to use:**
- Sending a complete, self-contained package to a client by email
- Archiving the mission with all its visual styling in a single file

## GPX — GPS Devices and Autopilots

**File extension:** `.gpx`

GPX (GPS Exchange Format) is compatible with Garmin, DJI and most autopilots that accept waypoint files.

**What's included:**
- A track per flight strip with ordered `<trkpt>` points
- Latitude, longitude and altitude for every waypoint

<Note>
  Altitude is exported in meters above the WGS84 ellipsoid, per the GPX specification.
</Note>

**When to use:**
- Loading waypoints into a GPS receiver or an autopilot
- Handing the flight plan to a ground control station that accepts GPX tracks

## CSV — Spreadsheets and Custom Pipelines

**File extension:** `.csv`

A comma-separated table of all waypoints in the flight plan.

**Column definitions:**

| Column | Type | Description |
|--------|------|-------------|
| `strip_id` | integer | Strip sequence number (1-based) |
| `waypoint_id` | integer | Waypoint index inside the strip |
| `latitude` | decimal | WGS84 latitude in decimal degrees |
| `longitude` | decimal | WGS84 longitude in decimal degrees |
| `altitude_m` | decimal | Altitude in meters (interpretation depends on the altitude mode) |
| `heading_deg` | decimal | Aircraft heading at this waypoint |

**When to use:**
- Importing into custom mission planning or post-processing workflows
- Scripting with Python, R or spreadsheets
- Generating internal reports in Excel

## GeoJSON — Web and API Integrations

**File extension:** `.geojson`

A JSON-based geographic format compatible with web mapping libraries (Leaflet, MapLibre, Mapbox) and modern GIS APIs.

**Typical structure:**
```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": { "type": "Polygon", "coordinates": [[[...]]] },
      "properties": { "type": "aoi" }
    },
    {
      "type": "Feature",
      "geometry": { "type": "LineString", "coordinates": [[...]] },
      "properties": { "type": "strip", "strip_id": 1, "heading": 45.0 }
    }
  ]
}
```

**When to use:**
- Embedding the mission in a web map
- Submitting to a REST API that accepts GeoJSON
- Storing mission geometry in a PostGIS database

## TXT — Pilot-Readable Waypoint List

**File extension:** `.txt`

A plain-text, human-readable list of every waypoint, formatted for reading by a pilot or copilot during flight.

```
HELIX FLIGHT PLAN — Project: Survey Zone A
Mission: M001 — Sensor: Sony A7R IV
Total strips: 12 — Total waypoints: 84

STRIP 01 (HDG 045 deg)
  WP 001  lat: 48.856614  lon: 2.352222  alt: 120 m
  WP 002  lat: 48.857032  lon: 2.353910  alt: 120 m
  ...
```

**When to use:**
- Printing a paper backup for the pilot
- Typing waypoints into a drone controller manually
- Quick review without a GIS tool

## XLSX — Excel Workbook

**File extension:** `.xlsx`

A structured Excel workbook with multiple sheets: mission parameters, strip list, waypoint list and QC summary.

**When to use:**
- Handing a formatted, print-ready worksheet to an operations team
- Running pivot tables or quick charts on the plan without any scripting

## Drone Waypoints

**File extension:** `.csv`

A dedicated CSV profile tailored to common drone controllers (column order and units match what most consumer / prosumer drones expect on import).

**When to use:**
- Loading a Helix plan into a drone controller without manual editing
- When the generic CSV export needs an additional reformatting step for the target drone

## Client View (HTML)

**File extension:** `.html`

A standalone HTML deliverable showing the AOI, the strips and the key mission statistics in a shareable, client-friendly layout. No Helix Flight Planner installation required — the recipient just opens the file in a browser.

**When to use:**
- Sharing a visual preview with a non-technical stakeholder
- Embedding the mission in a project page or an email handoff

## PDF Mission Report

**File extension:** `.pdf`

A professionally formatted multi-page PDF suitable for client delivery, regulatory submission or internal documentation. The PDF is generated through a preview dialog so you can pick which sections to include.

**Report sections:**

| Section | Contents |
|---------|----------|
| **Cover page** | Project name, mission name, date, operator, Helix logo |
| **Mission parameters** | Sensor, aircraft, altitude mode, altitude, overlap, sidelap, heading, GSD |
| **Statistics** | Frame count, strip count, total distance, flight time, coverage area |
| **Globe snapshot** | Top-down view of the AOI with flight strips overlaid |
| **QC maps** | Coverage, GSD, sidelap, HAGL, density and MTA overlays (when computed) |
| **Strip table** | Ordered strip list with waypoint count and heading |

**When to use:**
- Delivering a pre-flight brief to the client
- Submitting to a regulatory body for flight authorization
- Archiving mission documentation

## .helix — Native Helix Container

**File extension:** `.helix`

`.helix` is Helix Aero's native mission container. It is written by Helix Flight Planner on export and loaded locally by the Helix FMS software on the operator's machine.

**When to use:**
- Loading a mission into the Helix FMS from the operator's computer
- Archiving a complete mission for future reference
- Sharing a mission with another Helix Flight Planner user who will keep working on it

<Note>
  `.helix` is a proprietary Helix Aero format. Its internal structure is not publicly documented — treat it as an opaque container that only Helix Flight Planner and the Helix FMS are expected to read or write.
</Note>

## Export All

At the bottom of the export menu, **Export all** writes every supported format at once into a single folder of your choice. Useful when handing off a complete mission package to a client or to an operations team.
