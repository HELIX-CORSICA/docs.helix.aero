# Mission Planning

This guide covers the full mission planning workflow in Helix Flight Planner: from the dashboard and mission wizard through AOI drawing, parameter tuning, quality control and export.

## The Dashboard

When you open Helix Flight Planner, you land on the **Dashboard**. It lists all of your projects and missions in two columns:

- **Orphan missions** at the top — missions created without a parent project (sorted by most recent).
- **Project folders** below — expandable folders, each holding the missions that belong to that project.

The dashboard header lets you:

- Toggle between **grid view** (cards) and **list view** (compact rows).
- Sort by name, creation date or last modification.
- Filter by status and search missions by name.
- Open the **New Mission** wizard or create a new project.

A small sync indicator in the header reflects the current synchronization state of your workspace.

## The Mission Wizard

Click **New Mission** to open the wizard. It has six steps:

<Steps>
  <Step title="Project and mission name">
    Type the mission name and optionally attach it to a project. Missions created without a parent project appear in the orphan list.
  </Step>
  <Step title="Location">
    Search a location by name (geocoding) or click on the map to pin it. Optional — you can also skip this and draw the AOI later.
  </Step>
  <Step title="CRS (Coordinate Reference System)">
    Pick between **WGS84 / UTM auto** (default, zone computed from the pinned location) or a manual **EPSG** code.
  </Step>
  <Step title="DEM source">
    Choose the terrain source: **SRTM 30 m** (default, free, worldwide), **Copernicus GLO-30** (higher quality) or **None** (flat terrain, constant altitude).
  </Step>
  <Step title="Sensor">
    Browse the sensor catalog, select one, and pick a focal length if the sensor has multiple options.
  </Step>
  <Step title="Aircraft">
    Select the aircraft (helicopter, airplane or drone). The aircraft seeds default altitude, cruise speed and overlap for the mission.
  </Step>
</Steps>

Click **Create**. The 3D globe opens on the mission with the planning panel on the right.

## The 3D Globe

Helix Flight Planner uses a Cesium-based 3D globe embedded in the app. The viewer provides:

- **Satellite imagery** and global terrain.
- **Drawing toolbar** (left, under the layer button) with polygon AOI, corridor AOI and recenter.
- **Layer panel** (top-left) to toggle flight lines, waypoints, aero chart, GCPs and QC overlays.
- **Compass overlay** (top-right) showing heading and tilt.
- **Cursor info** in the status bar showing latitude, longitude and terrain elevation at the cursor.
- **Measurement tool** for distance, area and elevation profile.
- **Elevation profile overlay** generated from the DEM along any drawn line or flight strip.

### Defining the AOI

<Steps>
  <Step title="Draw a polygon">
    Click **Draw** in the drawing toolbar, click on the globe to place each vertex, and close the polygon by clicking the first point or pressing **Enter**. Drag a vertex to adjust it.
  </Step>
  <Step title="Or draw a corridor">
    Click **Draw corridor** to trace a linear corridor (useful for roads, rivers, pipelines, power lines). The planning engine will generate strips that follow the corridor's centerline.
  </Step>
  <Step title="Or import a KML / KMZ">
    From the dashboard or the mission, you can import an existing KML or KMZ file. The polygon loads straight onto the globe.
  </Step>
</Steps>

<Tip>
  For large or precise survey areas, importing a KML produced in a GIS tool is faster than drawing manually.
</Tip>

## The Planning Panel

The planning panel on the right of the mission view is organized into tabs:

| Tab | Purpose |
|-----|---------|
| **Parametres** | Altitude mode, altitude value, speed, heading, forward overlap, sidelap. |
| **Bandes** | Strip list and strip editor: rename, reverse, delete, reorder, run optimizer, add corrective strips. |
| **DEM** | DEM source, terrain cache state, elevation profile controls. |
| **Resume** | Mission statistics and QC summary ready for review or export. |

(Planning and globe state also surface contextual panels such as the QC alert banner and the elevation profile overlay.)

### Altitude Modes

Helix Flight Planner supports three altitude modes, switchable from the **Parametres** tab:

| Mode | Meaning |
|------|---------|
| **AMSL** | Absolute altitude above Mean Sea Level. Strips fly at a constant MSL altitude. |
| **iAMSL** | Initial AMSL — the typed altitude is interpreted as an MSL target computed at the mission's initial elevation. |
| **iAGL** | Initial AGL — the typed altitude is interpreted as height above ground level; strips follow the DEM to maintain constant terrain clearance. |

iAGL and iAMSL require a DEM source (SRTM or Copernicus) to be configured in the DEM tab. See the engine documentation for the detailed algorithm behind each mode.

### Flight Parameters

| Parameter | Description |
|-----------|-------------|
| **Altitude** | Target altitude (interpretation depends on the selected altitude mode). |
| **Target GSD** | Alternative: type a desired GSD and the app back-calculates the altitude. |
| **Speed** | Aircraft ground speed, used for flight time estimation and LiDAR point density. |
| **Heading** | Primary strip orientation in degrees. 0 = North-South. |
| **Forward overlap** | Overlap between consecutive frames along a strip (70-85% for photogrammetry). |
| **Sidelap** | Overlap between adjacent strips (60-70% for photogrammetry). |

All parameters recalculate strips in real time — no "Calculate" button.

## Reviewing Flight Strips

Once the AOI and parameters are set, strips appear on the globe. The layer panel lets you toggle:

- **Flight lines** — strip centerlines.
- **Waypoints** — the ordered waypoints that will be flown.
- **Aero chart** — local aeronautical chart overlay.
- **GCPs** — imported ground control points (when present).

### The Strip Editor (Bandes tab)

Open the **Bandes** tab in the planning panel to work on individual strips. Each strip can be:

- **Renamed** to a custom label.
- **Reversed** (flip start and end).
- **Deleted**.
- **Reordered** by drag and drop.

Two one-click tools are available at the top of the tab:

- **Optimizer** — reorders all primary and cross strips to minimize dead-leg distance.
- **Corrective strips** — adds extra strips over QC gaps detected by the coverage engine.

## Quality Control Overlays

Once a plan is computed, the QC engine evaluates it on a grid over the AOI. QC toggles appear in the layer panel as soon as results are available.

| Overlay | What it shows |
|---------|---------------|
| **Coverage** | Cells covered by at least one frame. Red cells mark gaps. |
| **GSD** | Effective GSD per cell (color-coded) — varies with terrain on iAGL missions. |
| **Sidelap** | Actual achieved sidelap per cell, accounting for strip geometry. |
| **HAGL** | Height above ground level per cell, to verify terrain clearance. |
| **Density** (LiDAR only) | Point density per square meter. |
| **MTA** (LiDAR only) | Multiple-Time-Around ambiguity risk zones. |

### QC Alerts

The app monitors QC values against configurable thresholds. When a value falls below the minimum:

- A **yellow alert banner** appears at the top of the mission view.
- The affected cells turn red on the corresponding QC overlay.

Typical alerts:
- **Coverage gap detected** — at least one cell has zero frames.
- **GSD exceeds threshold** — altitude is too high for the target resolution.
- **Sidelap below minimum** — strip spacing too wide.
- **HAGL out of range** — strips too low or too high above the terrain.

Adjust parameters, run the optimizer, or add corrective strips until the banner clears.

## Elevation Profile

From the measurement tool or from any strip, you can open the **Elevation profile overlay**. It plots terrain elevation along the selected line from the DEM, together with the planned flight altitude, so you can visually verify terrain clearance before exporting.

## Mission Summary (Resume tab)

The **Resume** tab shows the final computed values for the mission:

| Statistic | Description |
|-----------|-------------|
| Frame count | Total number of photos or LiDAR scan lines |
| Strip count | Number of parallel flight passes |
| Total distance | Sum of all flight line lengths |
| Flight time (est.) | Computed from distance and aircraft speed |
| GSD | Effective GSD at the configured altitude |
| Coverage area | AOI polygon area in hectares |

This is the tab you review right before export.

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Ctrl+N` | New mission |
| `Ctrl+S` | Save (manual save; auto-save also runs in the background) |
| `Ctrl+Z` / `Ctrl+Y` | Undo / redo on the flight plan |
| `Ctrl+C` / `Ctrl+V` | Copy / paste the AOI polygon |
| `Ctrl+E` | Open the export menu |
| `Ctrl + =` / `Ctrl + -` | Zoom the globe in / out |
| `Space` | Toggle top-down / free view |
| `F` | Recenter on the AOI |
| `M` | Start a distance measurement |
| `Delete` | Delete the selected element |
| `Escape` | Cancel the current action (drawing, measuring, modal) |
| `Tab` | Move between input fields in forms |

## Offline, Auto-save and Sync

- **Auto-save** runs in the background after every significant edit — your work is persisted locally so nothing is lost.
- **Offline mode** lets you keep planning when disconnected. All projects, missions and custom sensors remain available.
- **Sync** reconciles your workspace as soon as a connection is available; conflicts are resolved per mission.
- The license check re-verifies online within 7 days; beyond that, the app enters read-only mode until connectivity is restored.

## Export

When the mission is ready, click **Export** in the top toolbar to pick a format. See [Export Formats](/guides/export-formats) for a full description of each output.
