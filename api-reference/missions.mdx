---
title: "Missions"
description: "Create, retrieve, update and delete missions"
---

Missions are flight plans nested under a project. Each mission stores its AOI, sensor and aircraft references, flight plan parameters, computed strips and QC results.

<Info>
  **Auth:** every endpoint below requires `auth:api`. Access is gated by the `ProjectPolicy` (view) and `MissionPolicy` (update/delete). Mission routes are nested under their parent project.
</Info>

## Mission resource

<ResponseField name="id" type="string">Mission UUID.</ResponseField>
<ResponseField name="project_id" type="string">Parent project UUID.</ResponseField>
<ResponseField name="name" type="string">Mission name.</ResponseField>
<ResponseField name="description" type="string | null">Free-form description.</ResponseField>
<ResponseField name="sensor_id" type="integer | null">Referenced sensor (standard or custom).</ResponseField>
<ResponseField name="aircraft_id" type="integer | null">Referenced aircraft.</ResponseField>
<ResponseField name="flight_plan" type="object | null">Serialized flight plan payload (AOI, parameters, strips, stats, QC). Opaque to the API.</ResponseField>
<ResponseField name="status" type="string">One of `planned`, `in_progress`, `completed`, `cancelled`.</ResponseField>
<ResponseField name="mission_date" type="string | null">Planned mission date (`YYYY-MM-DD`).</ResponseField>
<ResponseField name="local_updated_at" type="string | null">Client-side timestamp used for last-write-wins conflict resolution.</ResponseField>
<ResponseField name="created_at" type="string">ISO 8601 creation timestamp.</ResponseField>
<ResponseField name="updated_at" type="string">ISO 8601 last update timestamp.</ResponseField>
<ResponseField name="deleted_at" type="string | null">Populated only when soft-deleted and still within the retention window.</ResponseField>

## POST /projects/\{project\}/missions

Create a new mission under a project.

<ParamField path="project" type="string" required>Parent project UUID.</ParamField>

<ParamField body="name" type="string" required>Mission name (max 255).</ParamField>
<ParamField body="description" type="string">Free-form description (max 5000).</ParamField>
<ParamField body="sensor_id" type="integer">Sensor foreign key.</ParamField>
<ParamField body="aircraft_id" type="integer">Aircraft foreign key.</ParamField>
<ParamField body="flight_plan" type="object">Serialized flight plan payload.</ParamField>
<ParamField body="status" type="string">One of `planned`, `in_progress`, `completed`, `cancelled`.</ParamField>
<ParamField body="mission_date" type="string">`YYYY-MM-DD` mission date.</ParamField>
<ParamField body="local_updated_at" type="string">Client-side timestamp for sync.</ParamField>

<RequestExample>
```json Body
{
  "name": "Zone Nord â€” 120m",
  "sensor_id": 87,
  "aircraft_id": 4,
  "flight_plan": {
    "aoi": { "type": "Polygon", "coordinates": [[[2.35, 48.85], [2.36, 48.85], [2.36, 48.86], [2.35, 48.86], [2.35, 48.85]]] },
    "parameters": { "altitude_m": 120, "overlap": 80, "sidelap": 70 }
  },
  "status": "planned",
  "mission_date": "2026-06-12"
}
```
</RequestExample>

### Errors

| Status | Meaning |
|--------|---------|
| `401` | Missing or expired JWT. |
| `403` | Not the owner of the parent project. |
| `404` | Project not found. |
| `422` | Validation failure. |

<ParamField path="project" type="string" required>Project UUID.</ParamField>

### Errors

| Status | Meaning |
|--------|---------|
| `401` | Missing or expired JWT. |
| `403` | Not the owner. |
| `404` | Mission or project not found. |

---

## PUT /projects/\{project\}/missions/\{mission\}

Update a mission. Every body field is optional; passing `flight_plan` replaces the stored object entirely.

### Errors

| Status | Meaning |
|--------|---------|
| `401` | Missing or expired JWT. |
| `403` | Not the owner of the mission. |
| `404` | Mission or project not found. |
| `422` | Validation failure. |

---

## DELETE /projects/\{project\}/missions/\{mission\}

Soft-delete a mission. The row is marked with `deleted_at` and pruned after 7 days by the `Prunable` trait. Propagate the tombstone to other clients via `POST /sync` before the prune window elapses.

<ResponseExample>
```http 204 No Content
```
</ResponseExample>

### Errors

| Status | Meaning |
|--------|---------|
| `401` | Missing or expired JWT. |
| `403` | Not the owner. |
| `404` | Mission or project not found. |
