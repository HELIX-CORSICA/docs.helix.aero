# Catalogue capteurs

Helix Flight Planner embarque un riche catalogue de **capteurs** (cameras, LiDAR, pushbroom) et d'**aeronefs** (helicopteres, avions, drones). Ce guide explique comment utiliser le catalogue du point de vue utilisateur — rechercher, filtrer, basculer entre vue grille et vue tableau, et creer vos propres entrees personnalisees.

Pour l'aspect mission (comment un capteur selectionne pilote le calepinage et le GSD) voir le [guide de planification](/guides/mission-planning).

## Ouvrir le catalogue

Ouvrez le **Catalogue** depuis la navigation de gauche. Il contient deux onglets :

- **Capteurs** — cameras, LiDAR et systemes pushbroom.
- **Aeronefs** — helicopteres, avions et drones.

Un petit indicateur de sync en en-tete indique si votre catalogue est a jour. Il se rafraichit automatiquement a la connexion et a chaque cycle de sync.

## Rechercher et filtrer

Les deux onglets partagent la meme barre de controles :

- **Champ de recherche** — correspond aux noms, fabricants et modeles de capteur ou d'aeronef. La requete est reinitialisee au changement d'onglet.
- **Chips de type** — pour l'onglet capteurs : **Tous**, **Camera**, **LiDAR**, **Pushbroom**. Pour l'onglet aeronefs : **Tous**, **Helicoptere**, **Avion**, **Drone**.
- **Bascule de vue** — passe entre vue grille et vue tableau.

Filtres et recherche se combinent — par exemple, `Recherche : "sony" + Type : Camera` affiche uniquement les cameras Sony.

## Vue grille vs vue tableau

La bascule de vue (en haut a droite du catalogue) commute entre deux dispositions :

<CardGroup cols={2}>
  <Card title="Vue grille" icon="grip">
    Une grille responsive de cartes — chaque carte affiche une photo du capteur ou de l'aeronef, son nom, son fabricant et ses specs cles. Ideale pour une navigation visuelle rapide.
  </Card>
  <Card title="Vue tableau" icon="table">
    Un tableau dense et triable — cliquez sur n'importe quel en-tete de colonne pour trier ascendant / descendant. Ideale pour comparer plusieurs elements sur une meme specification (focale, FOV, etc.).
  </Card>
</CardGroup>

La preference de vue est memorisee entre les sessions.

## Details du capteur

Cliquez sur une carte (ou une ligne) pour ouvrir la boite de dialogue **Details du capteur**. Elle affiche les specifications completes capturees dans le catalogue :

### Cameras

| Specification | Exemple |
|---------------|---------|
| Focale | 35 mm |
| Largeur / hauteur capteur | 35,9 x 24 mm |
| Taille pixel | 4,51 µm |
| Resolution image | 7952 x 5304 px |
| Type d'obturateur | Global / rolling |
| Recouvrement longitudinal recommande | 80 % |

### LiDAR

| Specification | Exemple |
|---------------|---------|
| FOV | 120 deg |
| Frequence d'impulsion | 600 kHz |
| Vitesse de balayage | 100 Hz |
| Divergence du faisceau | 0,5 mrad |
| Portee maximale | 500 m |
| Retours multiples | Jusqu'a 5 retours |

### Pushbroom

| Specification | Exemple |
|---------------|---------|
| Longueur de ligne | 12 000 px |
| Taille pixel | 6,5 µm |
| FOV | 40 deg |
| Frequence d'acquisition | 400 Hz |

La boite de dialogue propose aussi un raccourci **Utiliser pour une nouvelle mission** qui ouvre l'assistant de mission avec le capteur deja selectionne.

## Standards vs personnalises

Les capteurs et aeronefs existent en deux categories :

- **Standards** — curés par Helix Aero. Non modifiables ni supprimables localement ; si une specification est erronee, contactez le support. Les entrees standards sont partagees entre tous les utilisateurs de votre organisation et se mettent a jour automatiquement.
- **Personnalises** — crees par vous. Ils apparaissent a cote des entrees standards avec un petit badge **Personnalise** et sont limites a votre compte. Ils peuvent etre modifies ou supprimes a tout moment.

<Warning>
  Supprimer un capteur personnalise n'affecte pas les missions deja creees avec lui. Chaque mission conserve un instantane des specifications du capteur au moment de sa creation.
</Warning>

## Creer un capteur personnalise

Cliquez sur **Nouveau capteur** en haut du catalogue (ou sur le bouton flottant du panneau capteur dans une mission). Un formulaire s'ouvre :

<Steps>
  <Step title="Choisir le type de capteur">
    Camera, LiDAR ou Pushbroom.
  </Step>
  <Step title="Renseigner l'identite">
    Nom, fabricant et modele.
  </Step>
  <Step title="Renseigner les specifications techniques">
    Les champs affiches dependent du type (voir les tableaux de specifications ci-dessus). Les champs requis sont marques ; l'application refuse l'enregistrement tant qu'ils ne sont pas remplis.
  </Step>
  <Step title="Enregistrer">
    Le capteur est enregistre localement et synchronise avec votre compte au prochain cycle de sync. Il est immediatement disponible dans l'assistant de mission.
  </Step>
</Steps>

<Tip>
  Pour les cameras, si vous connaissez la taille de pixel et la resolution image, les dimensions du capteur sont derivees automatiquement. Vous pouvez aussi saisir directement les dimensions et la taille de pixel est calculee.
</Tip>

## Creer un aeronef personnalise

L'onglet aeronefs supporte le meme flux. Les aeronefs personnalises permettent de definir des valeurs par defaut d'altitude, vitesse et recouvrement qui serviront a initialiser les nouvelles missions. L'etape aeronef de l'assistant liste les aeronefs personnalises a cote du catalogue standard.

## Modifier et supprimer

Seules les entrees personnalisees sont modifiables. Cliquez sur le petit menu de la carte (ou de la ligne) pour acceder a **Modifier** et **Supprimer**. Les entrees standards affichent un cadenas — pour proposer des corrections, contactez le support Helix Aero.
