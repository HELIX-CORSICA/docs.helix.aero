# Formats d'export

Helix Flight Planner peut exporter un plan de vol termine dans de nombreux formats. Ouvrez le menu **Exporter** dans la barre d'outils de la vue mission (ou appuyez sur `Ctrl+E`) pour en choisir un. Le menu est regroupe en :

- **Plan de vol** — formats geometriques / de donnees pour outils externes (KML, KMZ, GPX, CSV, GeoJSON, XLSX).
- **Vue client** — un livrable HTML pour les parties prenantes non techniques.
- **Waypoints drone** — un profil CSV adapte aux controleurs de drone.
- **Rapport PDF** — un rapport multi-pages imprimable.
- **Fichier Helix natif** — un conteneur opaque a charger localement dans le FMS Helix.

## KML — Google Earth et SIG

**Extension :** `.kml`

KML (Keyhole Markup Language) est le format geographique standard utilise par Google Earth et la plupart des outils SIG.

**Contenu :**
- Polygone de la zone d'interet
- Lignes de vol sous forme de `LineString`
- Empreintes camera (une par image)
- Etiquettes de bande avec cap et numero de sequence

**Quand l'utiliser :**
- Partager la mission avec le pilote pour une revision dans Google Earth
- Importer dans QGIS, ArcGIS ou outil SIG pour analyse complementaire
- Soumettre une zone de vol visuelle aux autorites aeriennes

## KMZ — KML zippe

**Extension :** `.kmz`

Archive KML compressee avec styles et overlays raster embarques. KMZ se transporte mieux que KML quand l'export contient des styles ou des heatmaps QC.

**Quand l'utiliser :**
- Envoyer un paquet complet et autonome a un client par mail
- Archiver la mission avec tout son style visuel dans un seul fichier

## GPX — GPS et autopilotes

**Extension :** `.gpx`

GPX (GPS Exchange Format) est compatible avec Garmin, DJI et la plupart des autopilotes qui acceptent des fichiers de waypoints.

**Contenu :**
- Une trace par bande avec points `<trkpt>` ordonnes
- Latitude, longitude et altitude pour chaque waypoint

<Note>
  L'altitude est exportee en metres au-dessus de l'ellipsoide WGS84, selon la specification GPX.
</Note>

**Quand l'utiliser :**
- Charger des waypoints dans un recepteur GPS ou un autopilote
- Remettre le plan a une station de controle au sol acceptant les traces GPX

## CSV — Tableurs et pipelines

**Extension :** `.csv`

Table CSV de tous les waypoints du plan de vol.

**Definition des colonnes :**

| Colonne | Type | Description |
|---------|------|-------------|
| `strip_id` | entier | Numero de sequence de la bande (base 1) |
| `waypoint_id` | entier | Index du waypoint dans la bande |
| `latitude` | decimal | Latitude WGS84 en degres decimaux |
| `longitude` | decimal | Longitude WGS84 en degres decimaux |
| `altitude_m` | decimal | Altitude en metres (interpretation selon le mode d'altitude) |
| `heading_deg` | decimal | Cap de l'aeronef a ce waypoint |

**Quand l'utiliser :**
- Importer dans des workflows personnalises de planification ou post-traitement
- Scripter avec Python, R ou tableurs
- Generer des rapports internes dans Excel

## GeoJSON — Web et API

**Extension :** `.geojson`

Format geographique JSON compatible avec les bibliotheques de cartographie web (Leaflet, MapLibre, Mapbox) et les API SIG modernes.

**Structure typique :**
```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": { "type": "Polygon", "coordinates": [[[...]]] },
      "properties": { "type": "aoi" }
    },
    {
      "type": "Feature",
      "geometry": { "type": "LineString", "coordinates": [[...]] },
      "properties": { "type": "strip", "strip_id": 1, "heading": 45.0 }
    }
  ]
}
```

**Quand l'utiliser :**
- Integrer la mission dans une carte web
- Soumettre a une API REST acceptant GeoJSON
- Stocker la geometrie dans une base PostGIS

## TXT — Liste de waypoints lisible

**Extension :** `.txt`

Liste texte, lisible, de tous les waypoints, formatee pour etre consultee par un pilote ou copilote en vol.

```
HELIX FLIGHT PLAN — Projet : Zone A
Mission : M001 — Capteur : Sony A7R IV
Total bandes : 12 — Total waypoints : 84

BANDE 01 (CAP 045 deg)
  WP 001  lat: 48.856614  lon: 2.352222  alt: 120 m
  WP 002  lat: 48.857032  lon: 2.353910  alt: 120 m
  ...
```

**Quand l'utiliser :**
- Imprimer un backup papier pour le pilote
- Saisir les waypoints manuellement dans un controleur de drone
- Relecture rapide sans outil SIG

## XLSX — Classeur Excel

**Extension :** `.xlsx`

Classeur Excel structure avec plusieurs feuilles : parametres de mission, liste des bandes, liste des waypoints et synthese QC.

**Quand l'utiliser :**
- Remettre une feuille formatee prete a imprimer a une equipe operationnelle
- Executer des tableaux croises ou des graphiques rapides sans scripting

## Waypoints drone

**Extension :** `.csv`

Profil CSV specifique adapte aux controleurs de drone courants (ordre des colonnes et unites conformes a ce que la plupart des drones grand public / prosumer attendent a l'import).

**Quand l'utiliser :**
- Charger un plan Helix dans un controleur de drone sans retravail manuel
- Quand le CSV generique necessite un reformatage supplementaire pour le drone cible

## Vue client (HTML)

**Extension :** `.html`

Livrable HTML autonome montrant l'AOI, les bandes et les statistiques cles de la mission dans une mise en page client. Aucune installation de Helix Flight Planner requise — le destinataire ouvre le fichier dans un navigateur.

**Quand l'utiliser :**
- Partager un apercu visuel avec une partie prenante non technique
- Integrer la mission dans une page projet ou dans un mail

## Rapport PDF

**Extension :** `.pdf`

Rapport PDF multi-pages au formatage professionnel, adapte a la livraison client, la soumission reglementaire ou la documentation interne. Le PDF est genere via une boite de dialogue d'apercu qui permet de choisir les sections a inclure.

**Sections du rapport :**

| Section | Contenu |
|---------|---------|
| **Page de couverture** | Projet, mission, date, operateur, logo Helix |
| **Parametres de mission** | Capteur, aeronef, mode et valeur d'altitude, recouvrement, laterale, cap, GSD |
| **Statistiques** | Nombre d'images, nombre de bandes, distance totale, temps de vol, surface couverte |
| **Capture du globe** | Vue zenithale de l'AOI avec les bandes en superposition |
| **Cartes QC** | Couverture, GSD, laterale, HAGL, densite et MTA (si calcules) |
| **Tableau des bandes** | Liste ordonnee des bandes avec nombre de waypoints et cap |

**Quand l'utiliser :**
- Livrer un brief pre-vol au client
- Soumettre a une autorite reglementaire pour autorisation de vol
- Archiver la documentation de mission

## .helix — Conteneur natif Helix

**Extension :** `.helix`

`.helix` est le conteneur de mission natif de Helix Aero. Il est produit par Helix Flight Planner a l'export et charge localement dans le logiciel FMS Helix sur la machine de l'operateur.

**Quand l'utiliser :**
- Charger une mission dans le FMS Helix depuis l'ordinateur de l'operateur
- Archiver une mission complete pour reference future
- Partager une mission avec un autre utilisateur Helix Flight Planner qui continuera a travailler dessus

<Note>
  `.helix` est un format proprietaire Helix Aero. Sa structure interne n'est pas documentee publiquement — considerez-le comme un conteneur opaque que seuls Helix Flight Planner et le FMS Helix savent lire ou ecrire.
</Note>

## Tout exporter

En bas du menu, **Tout exporter** ecrit chaque format supporte dans un dossier unique de votre choix. Pratique pour livrer un package complet a un client ou a une equipe operationnelle.
